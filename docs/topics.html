 CAN_FuncsUser defined functions for interacting with CAN bus. These effectively wrap the HAL functions and use FreeRTOS mutexes to guarantee no race conditions. CAN Function docs begin on page 87 of UM1725!!
 CAN_Motor_FuncsFunctions for interacting with motor controllers via the CAN bus. Each function's parameter is a pointer to a motor packet info struct, and the functions assume they've been initialized to a default. It is from here (pmpi->ControllerInfo->ID) that the motor controller's CAN ID will be obtained. Additionally, all functions will be blocking until either the controller responds, or the CCU_MOTOR_TIMEOUT times out
